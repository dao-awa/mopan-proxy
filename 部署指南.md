## 一、MopanProxy 部署

### 1.1 本地编译

```bash
cd /d/project/移动云盘挂载
GOOS=linux GOARCH=amd64 go build -o mopan-proxy-linux ./cmd/mopan-proxy/
```

### 1.2 上传到服务器

```bash
# 创建目录
ssh root@154.19.43.9 "mkdir -p /opt/mopan-proxy/data"

# 上传二进制
scp mopan-proxy-linux root@154.19.43.9:/opt/mopan-proxy/mopan-proxy

# 设置权限
ssh root@154.19.43.9 "chmod +x /opt/mopan-proxy/mopan-proxy"
```

### 1.3 服务器配置文件

路径: `/opt/mopan-proxy/config.yaml`

```yaml
port: 9090
host: "0.0.0.0"

# 和彩云 Token（从浏览器 F12 → Network → Authorization 头获取）
authorization: "Basic 你的base64token"

# Web UI 登录保护
auth:
  enabled: true
  username: "admin"
  password: "你的登录密码"

# WebDAV
webdav:
  enabled: true
  port: 9091
  username: "你的webdav用户名"
  password: "你的webdav密码"

# 加密
crypto:
  enabled: true
  password: "你的加密密码"
  iterations: 100000

# 存储
storage:
  cache_dir: "./data/cache"
  meta_path: "./data/metadata.db"
  max_upload: 10737418240
  root_folder: "/MopanProxy"
```

⚠️ 配置文件包含敏感信息，权限必须 0600：
```bash
chmod 600 /opt/mopan-proxy/config.yaml
```

### 1.4 Systemd 服务

```bash
cat > /etc/systemd/system/mopan-proxy.service << 'EOF'
[Unit]
Description=MopanProxy - 和彩云加密代理
After=network.target

[Service]
Type=simple
ExecStart=/opt/mopan-proxy/mopan-proxy -config /opt/mopan-proxy/config.yaml
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable mopan-proxy
systemctl start mopan-proxy
```

### 1.5 验证

```bash
# 检查状态
systemctl status mopan-proxy

# 测试本地端口
curl -s http://127.0.0.1:9090/api/status

# 查看日志
journalctl -u mopan-proxy -f
```

## 二、Cloudflare Origin 证书

### 2.1 创建证书

1. 登录 Cloudflare Dashboard
2. 选择你的域名 (96110.li)
3. SSL/TLS → Origin Server → Create Certificate
4. 保持默认设置（RSA, 15年）
5. 记录生成的 PEM 和 KEY 内容

### 2.2 上传证书到服务器

**方法一：直接在服务器生成（推荐）**

```bash
ssh root@154.19.43.9
mkdir -p /etc/nginx/certs
chmod 700 /etc/nginx/certs
```

然后在 Cloudflare Dashboard 复制证书内容，用 SSH 工具上传。

**方法二：本地上传**

```bash
# 本地先用 Python 编码（避免 base64 换行问题）
python3 -c "
import base64
with open('webdav.96110.li.pem', 'rb') as f:
    print(base64.b64encode(f.read()).decode())
" > cert_b64.txt

# 上传到服务器
scp cert_b64.txt root@154.19.43.9:/tmp/
scp key_b64.txt root@154.19.43.9:/tmp/

# 在服务器上解码
ssh root@154.19.43.9 << 'DECODE'
mkdir -p /etc/nginx/certs
chmod 700 /etc/nginx/certs
python3 -c "
import base64
with open('/tmp/cert_b64.txt') as f:
    cert = base64.b64decode(f.read().strip())
with open('/etc/nginx/certs/webdav.96110.li.pem', 'wb') as out:
    out.write(cert)
with open('/tmp/key_b64.txt') as f:
    key = base64.b64decode(f.read().strip())
with open('/etc/nginx/certs/webdav.96110.li.key', 'wb') as out:
    out.write(key)
"
chmod 600 /etc/nginx/certs/*.key
DECODE
```

### 2.3 两个域名的证书

需要创建两套证书：
- `webdav.96110.li.pem` / `webdav.96110.li.key`
- `qb.96110.li.pem` / `qb.96110.li.key`

## 三、Nginx 配置

### 3.1 安装 Nginx

```bash
apt-get update
apt-get install -y nginx
```

### 3.2 MopanProxy 站点 (webdav.96110.li)

```bash
cat > /etc/nginx/sites-enabled/webdav.96110.li << 'EOF'
server {
    listen 443 ssl http2;
    server_name webdav.96110.li;

    ssl_certificate     /etc/nginx/certs/webdav.96110.li.pem;
    ssl_certificate_key /etc/nginx/certs/webdav.96110.li.key;

    # 大文件上传支持
    client_max_body_size 10g;
    client_body_buffer_size 128k;

    # 超时设置（WebDAV 大文件传输）
    proxy_read_timeout 600s;
    proxy_send_timeout 600s;
    send_timeout 600s;

    # 禁用缓冲（流式传输）
    proxy_buffering off;

    # WebDAV 代理到 9091
    location /dav/ {
        proxy_pass http://127.0.0.1:9091;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_pass_header Authorization;
    }

    # Web UI 代理到 9090
    location / {
        proxy_pass http://127.0.0.1:9090;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name webdav.96110.li;
    return 301 https://$host$request_uri;
}
EOF
```

### 3.3 qBittorrent 站点 (qb.96110.li)

```bash
cat > /etc/nginx/sites-enabled/qb.96110.li << 'EOF'
server {
    listen 443 ssl http2;
    server_name qb.96110.li;

    ssl_certificate     /etc/nginx/certs/qb.96110.li.pem;
    ssl_certificate_key /etc/nginx/certs/qb.96110.li.key;

    client_max_body_size 100m;

    # 防止 qBittorrent 长轮询导致超时
    proxy_read_timeout 300s;
    proxy_send_timeout 300s;
    proxy_connect_timeout 10s;

    location / {
        proxy_pass http://127.0.0.1:9092;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name qb.96110.li;
    return 301 https://$host$request_uri;
}
EOF
```

### 3.4 删除默认配置 & 启用

```bash
# 删除默认站点（避免冲突）
rm -f /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 重载
nginx -s reload
```

### 3.5 Cloudflare SSL 设置

必须在 Cloudflare Dashboard 设置 SSL/TLS 为 **Full (strict)**，否则会 522。

## 五、rclone 挂载（云盘 → 本地）

### 5.1 安装

```bash
apt-get install -y rclone
```

### 5.2 配置

```bash
mkdir -p ~/.config/rclone
PASS=$(rclone obscure "你的webdav密码")

cat > ~/.config/rclone/rclone.conf << EOF
[yidong]
type = webdav
url = http://127.0.0.1:9091/dav/
vendor = other
user = 你的webdav用户名
pass = $PASS

# 2GB VFS 缓存（做种读取用）
vfs_cache_mode = full
vfs_cache_max_size = 2147483648
vfs_cache_max_age = 168h
vfs_read_chunk_size = 16M
vfs_read_chunk_size_limit = 512M
dir_cache_time = 5m
chunk_size = 16M
vfs_cache_poll_interval = 30s
EOF
```

### 5.3 手动挂载测试

```bash
mkdir -p /mnt/yidong

# 挂载
rclone mount yidong: /mnt/yidong --daemon --allow-other

# 验证挂载
df -h /mnt/yidong

# 测试写入
echo "test" > /mnt/yidong/test.txt
ls /mnt/yidong/
rm /mnt/yidong/test.txt
```

### 5.4 开机自动挂载

```bash
cat >> /etc/rc.local << 'BOOT'
# Mount yidong WebDAV (optimized)
/usr/bin/rclone mount yidong: /mnt/yidong --daemon --allow-other 2>/dev/null
BOOT
chmod +x /etc/rc.local
```

### 5.5 手动卸载/挂载

```bash
# 卸载
fusermount -u /mnt/yidong

# 重新挂载
rclone mount yidong: /mnt/yidong --daemon --allow-other
```

## 六、云盘目录结构

### 6.1 在 Web UI 中创建 /qb 文件夹

1. 访问 https://webdav.96110.li
2. 登录
3. 点击 "新建文件夹"
4. 输入名称: `qb`

### 6.2 或者通过 API 创建

```bash
# 登录获取 session
COOKIE=$(curl -sk -c - -X POST https://webdav.96110.li/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"你的密码"}' \
  --connect-timeout 10 | grep session_id | awk '{print $NF}')

# 创建 qb 文件夹
curl -sk -b "session_id=$COOKIE" -X POST https://webdav.96110.li/api/mkdir \
  -H "Content-Type: application/json" \
  -H "X-Requested-With: XMLHttpRequest" \
  -d '{"parent_id":"/","name":"qb"}' \
  --connect-timeout 10
```

### 6.3 最终云盘结构

```
/（根目录）
├── 种子1/          ← 普通文件
├── 种子2/
└── qb/             ← qBittorrent 下载目录

    └── 下载2/
```

## 十、常见问题

### 522 Connection Timed Out

**原因**: Cloudflare 无法连接到源站

**解决**:
```bash
# 检查服务状态
systemctl status mopan-proxy
systemctl status nginx

# 重启服务
systemctl restart mopan-proxy
systemctl restart nginx
```

### rclone 挂载后 qb 下载失败

**原因**: rclone 缓存配置问题

**解决**:
```bash
# 检查挂载
df -h /mnt/yidong
mount | grep yidong

# 重新挂载
fusermount -u /mnt/yidong
rclone mount yidong: /mnt/yidong --daemon --allow-other
```

### qBittorrent 密码错误

**原因**: 密码未正确配置

**解决**:
```bash
# 查看启动日志获取临时密码
journalctl -u qbittorrent -n 50 | grep -i password
```

### WebDAV 无法连接

**原因**: 认证问题或端口未开放

**解决**:
```bash
# 测试本地端口
curl -u admin:password http://127.0.0.1:9091/dav/

# 检查防火墙
ufw status
```

## 十二、安全提醒

1. **加密密码**：丢失后无法解密文件，务必备份到安全位置
2. **config.yaml**：包含 Token 和密码，权限必须 0600
3. **WebDAV 端口 9091**：不要直接暴露到公网，必须通过 nginx 代理
4. **qBittorrent 密码**：首次启动后在 Web UI 中修改
5. **rclone 缓存**：2GB 上限，超出自动淘汰旧数据
6. **Cloudflare SSL**：必须设为 Full (strict)，否则会 522
7. **Token 过期**：和彩云 Token 约 30 天过期，需重新获取
8. **数据库备份**：定期备份 `/opt/mopan-proxy/data/metadata.db`
