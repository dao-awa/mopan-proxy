# MopanProxy 服务器部署指南

## 环境要求

- OS: Ubuntu/Debian (已测试: Debian 12)
- Go: 1.25+
- Nginx
- Cloudflare (SSL + CDN)
- 域名: 你的域名

## 一、编译

```bash
# 本地编译 Linux 二进制
cd /path/to/mopan-proxy
GOOS=linux GOARCH=amd64 go build -o mopan-proxy-linux ./cmd/mopan-proxy/
```

## 二、上传到服务器

```bash
# 创建目录
ssh vps "mkdir -p /opt/mopan-proxy/data"

# 上传二进制
scp mopan-proxy-linux vps:/opt/mopan-proxy/mopan-proxy

# 上传配置
scp config.example.yaml vps:/opt/mopan-proxy/config.yaml

# 设置权限
ssh vps "chmod +x /opt/mopan-proxy/mopan-proxy"
```

## 三、配置文件

编辑服务器上的 `/opt/mopan-proxy/config.yaml`：

```yaml
port: 9090
host: "0.0.0.0"

# 和彩云 Token（从浏览器 F12 获取）
authorization: "Basic base64(pc:手机号:token)"

# Web UI 登录保护（默认开启）
auth:
  enabled: true
  username: "admin"
  password: "你的密码"    # ⚠️ 必须修改

# WebDAV
webdav:
  enabled: true
  port: 9091
  username: "你的用户名"
  password: "你的密码"    # ⚠️ 必须修改

# 加密
crypto:
  enabled: true
  password: "你的加密密码"  # ⚠️ 必须修改，丢失无法解密
  iterations: 100000

# 存储
storage:
  cache_dir: "./data/cache"
  meta_path: "./data/metadata.db"
  max_upload: 10737418240
  root_folder: "/MopanProxy"
```

## 四、Systemd 服务

```bash
cat > /etc/systemd/system/mopan-proxy.service << 'EOF'
[Unit]
Description=MopanProxy - 和彩云加密代理
After=network.target

[Service]
Type=simple
ExecStart=/opt/mopan-proxy/mopan-proxy -config /opt/mopan-proxy/config.yaml
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable mopan-proxy
systemctl start mopan-proxy
systemctl status mopan-proxy
```

## 五、Nginx 配置

### 5.1 SSL 证书

使用 Cloudflare Origin 证书（在 Cloudflare Dashboard → SSL/TLS → Origin Server 创建）：

```bash
mkdir -p /etc/nginx/certs
# 上传证书文件
scp your-domain.pem vps:/etc/nginx/certs/your-domain.pem
scp your-domain.key vps:/etc/nginx/certs/your-domain.key
chmod 600 /etc/nginx/certs/your-domain.key
```

### 5.2 Nginx 站点配置

```bash
cat > /etc/nginx/sites-enabled/your-domain << 'EOF'
server {
    listen 443 ssl http2;
    server_name your-domain;

    ssl_certificate     /etc/nginx/certs/your-domain.pem;
    ssl_certificate_key /etc/nginx/certs/your-domain.key;

    # 大文件上传支持
    client_max_body_size 10g;
    client_body_buffer_size 128k;

    # 超时设置（WebDAV 大文件传输）
    proxy_read_timeout 600s;
    proxy_send_timeout 600s;
    send_timeout 600s;

    # 禁用缓冲（流式传输）
    proxy_buffering off;

    # WebDAV 代理到 9091
    location /dav/ {
        proxy_pass http://127.0.0.1:9091;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_pass_header Authorization;
    }

    # Web UI 代理到 9090
    location / {
        proxy_pass http://127.0.0.1:9090;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name your-domain;
    return 301 https://$host$request_uri;
}
EOF

nginx -t && nginx -s reload
```

## 六、qBittorrent 部署

### 6.1 安装

```bash
apt-get update
apt-get install -y qbittorrent-nox
```

### 6.2 Cloudflare 证书

在 Cloudflare Dashboard 为 `qb.your-domain` 创建 Origin 证书，上传到 `/etc/nginx/certs/`。

### 6.3 Nginx 配置

```bash
cat > /etc/nginx/sites-enabled/qb.your-domain << 'EOF'
server {
    listen 443 ssl http2;
    server_name qb.your-domain;

    ssl_certificate     /etc/nginx/certs/qb.your-domain.pem;
    ssl_certificate_key /etc/nginx/certs/qb.your-domain.key;

    client_max_body_size 100m;

    # 防止 qBittorrent 长轮询导致超时
    proxy_read_timeout 300s;
    proxy_send_timeout 300s;
    proxy_connect_timeout 10s;

    location / {
        proxy_pass http://127.0.0.1:9092;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name qb.your-domain;
    return 301 https://$host$request_uri;
}
EOF

nginx -t && nginx -s reload
```

### 6.4 Systemd 服务

```bash
cat > /etc/systemd/system/qbittorrent.service << 'EOF'
[Unit]
Description=qBittorrent BitTorrent client
After=network.target

[Service]
Type=simple
User=root
ExecStart=/usr/bin/qbittorrent-nox
Restart=on-failure
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable qbittorrent
systemctl start qbittorrent
```

### 6.5 配置下载路径

```bash
# 配置文件位置
/root/.config/qBittorrent/qBittorrent.conf

# 关键配置项
WebUI\Port=9092
Downloads\SavePath=/mnt/yidong/qb
Downloads\TempPath=/mnt/yidong/qb/.qb-temp
```

⚠️ 修改配置前必须停止服务，修改后重启生效。

## 七、rclone 挂载（云盘 → 本地）

### 7.1 安装

```bash
apt-get install -y rclone
```

### 7.2 配置

```bash
mkdir -p ~/.config/rclone
PASS=$(rclone obscure "你的WebDAV密码")

cat > ~/.config/rclone/rclone.conf << EOF
[yidong]
type = webdav
url = http://127.0.0.1:9091/dav/
vendor = other
user = 你的WebDAV用户名
pass = $PASS

# 2GB VFS 缓存（做种读取用）
vfs_cache_mode = full
vfs_cache_max_size = 2147483648
vfs_cache_max_age = 168h
vfs_read_chunk_size = 16M
vfs_read_chunk_size_limit = 512M
dir_cache_time = 5m
chunk_size = 16M
vfs_cache_poll_interval = 30s
EOF
```

### 7.3 挂载

```bash
mkdir -p /mnt/yidong

# 手动挂载测试
rclone mount yidong: /mnt/yidong --daemon --allow-other

# 开机自动挂载
cat >> /etc/rc.local << 'BOOT'
# Mount yidong WebDAV (optimized)
/usr/bin/rclone mount yidong: /mnt/yidong --daemon --allow-other 2>/dev/null
BOOT
chmod +x /etc/rc.local
```

### 7.4 验证

```bash
# 检查挂载
df -h /mnt/yidong

# 测试写入（应该显示正确文件名，不是 E_ 加密名）
echo "test" > /mnt/yidong/test.txt
ls /mnt/yidong/
rm /mnt/yidong/test.txt
```

## 八、云盘目录结构

在 MopanProxy Web UI 中创建 `/qb` 文件夹：

```bash
# 通过 API 创建（保留元数据记录）
COOKIE=$(curl -sk -c - -X POST https://your-domain/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"你的密码"}' \
  --connect-timeout 10 | grep session_id | awk '{print $NF}')

curl -sk -b "session_id=$COOKIE" -X POST https://your-domain/api/mkdir \
  -H "Content-Type: application/json" \
  -H "X-Requested-With: XMLHttpRequest" \
  -d '{"parent_id":"/","name":"qb"}' \
  --connect-timeout 10
```

最终云盘结构：
```
/（根目录）
└── qb/          ← qBittorrent 下载目录
    ├── 种子1/
    └── 种子2/
```

## 九、自动同步脚本（可选）

如果需要从本地临时目录同步到云盘：

```bash
cat > /usr/local/bin/qb-sync-cloud.sh << 'SCRIPT'
#!/bin/bash
SRC="/opt/qb-downloads"
LOG="/var/log/qb-sync.log"
API="https://your-domain"
COOKIES="/tmp/qb-sync-cookies.txt"

login() {
    curl -sk -c "$COOKIES" -X POST "$API/api/login" \
        -H "Content-Type: application/json" \
        -d '{"username":"admin","password":"你的密码"}' \
        --connect-timeout 10 -o /dev/null 2>/dev/null
}

upload_file() {
    curl -sk -b "$COOKIES" -X POST "$API/api/upload" \
        -F "file=@$1" -F "path=/" \
        -H "X-Requested-With: XMLHttpRequest" \
        --connect-timeout 300 --max-time 600 -o /dev/null 2>/dev/null
}

login || { echo "$(date) login failed" >> "$LOG"; exit 1; }

for item in "$SRC"/*; do
    [ -e "$item" ] || continue
    name=$(basename "$item")
    [ "$name" = ".tmp" ] && continue
    [ -d "$item" ] && continue
    echo "$(date) Uploading: $name" >> "$LOG"
    upload_file "$item" && rm -f "$item" && echo "$(date) Done: $name" >> "$LOG"
done
SCRIPT

chmod +x /usr/local/bin/qb-sync-cloud.sh
(crontab -l 2>/dev/null | grep -v qb-sync; echo "*/5 * * * * /usr/local/bin/qb-sync-cloud.sh >> /var/log/qb-sync.log 2>&1") | crontab -
```

## 十一、注意事项

1. **加密密码**：丢失后无法解密文件，务必备份
2. **config.yaml**：包含 Token 和密码，权限必须 0600
3. **WebDAV 端口 9091**：不要直接暴露到公网，通过 nginx 代理
4. **qBittorrent 密码**：首次启动后在 Web UI 中修改
5. **rclone 缓存**：2GB 上限，超出自动淘汰旧数据
6. **Cloudflare SSL**：必须设为 Full（非 Flexible），否则会 522
7. **Token 过期**：和彩云 Token 约 30 天过期，需重新获取
