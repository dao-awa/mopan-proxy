# MopanProxy 完整项目文档

> 和彩云（中国移动云盘 yun.139.com）**加密代理服务器**
> 提供 Web UI 文件管理 + WebDAV 网络驱动器 + AES-256-GCM 端到端加密
> **文件名 + 文件夹名 + 文件内容全部加密，云端无法辨别任何信息**

### 2.2 数据流

#### 上传流程
```
用户上传: 文件名 "report.pdf", 内容 "Hello World"
    ↓
MopanProxy 接收
    ↓
名称加密: "report.pdf" → "E_50db1161a60377ce...bin"
    ↓
内容加密: "Hello World" → [12B nonce][密文][16B GCM tag]
    ↓
计算 SHA256(密文)  ←→  用于秒传/去重
    ↓
POST /hcy/file/create  ←→  创建文件记录 (云端名: E_xxx.bin)
    ↓
PUT uploadUrl          ←→  上传密文到 COS
    ↓
POST /hcy/file/complete ←→  确认上传完成
    ↓
INSERT INTO SQLite      ←→  保存映射 (E_xxx.bin → report.pdf)
```

#### 下载流程
```
用户请求下载 "report.pdf"
    ↓
SQLite 查询: report.pdf → E_50db1161a60377ce...bin (cloud_file_id)
    ↓
POST /hcy/file/getDownloadUrl  ←→  获取 CDN URL
    ↓
GET CDN URL                     ←→  下载密文
    ↓
AES-256-GCM 解密内容            ←→  使用存储的 nonce
    ↓
返回明文给用户
```

### 2.3 核心模块

| 模块 | 文件 | 职责 |
|------|------|------|
| 加密引擎 | `internal/crypto/aes256gcm.go` | 内容加密 + 名称加密 |

| API 客户端 | `internal/mopan/client.go` | 和彩云 API 封装 |

| 配置 | `internal/config/config.go` | YAML 配置加载/保存 |
| 入口 | `cmd/mopan-proxy/main.go` | 优雅关闭 + 信号处理 |

## 四、配置说明

### 4.1 config.yaml 完整示例

```yaml
# 服务器
port: 9090
host: "0.0.0.0"
debug: false

# 和彩云认证
account: "13800138000"
authorization: "Basic cGM6MTk4..."

# WebDAV
webdav:
  enabled: true
  port: 9091
  username: "admin"
  password: "admin123"

# 加密
crypto:
  enabled: true                # 是否启用加密
  password: "your-password"    # 加密密码（⚠️ 请修改默认值！）
  salt_hex: ""                 # Salt（自动生成，勿手动修改）
  iterations: 100000           # PBKDF2 迭代次数

# 存储
storage:
  cache_dir: "./data/cache"    # 临时缓存目录
  meta_path: "./data/metadata.db"  # SQLite 数据库路径
  max_upload: 10737418240      # 最大上传大小 (10GB)
  root_folder: "/MopanProxy"   # 和彩云专用根目录

# Token 刷新
refresh:
  enabled: true
```

### 4.2 关键配置项

#### crypto.password
- **重要**: 加密密码，用于派生 AES-256 密钥
- **默认值**: `mopan-default-password`（请务必修改！）
- **丢失后果**: 已加密的文件**无法解密**

#### crypto.salt_hex
- **自动生成**: 首次启动时生成 16 字节随机 salt
- **不要修改**: 修改 salt 等于更换密钥

#### storage.root_folder
- 在和彩云上创建专用目录，所有文件存储在此
- 与云盘中的其他文件完全隔离

### 4.3 首次启动流程

```
1. 检测 config.yaml → 不存在则创建默认配置
2. 自动生成 salt（如为空）
3. PBKDF2 派生 AES-256 密钥
4. 初始化 SQLite (WAL 模式)
5. 查找/创建根目录 (/MopanProxy)
6. 启动 Web UI (:9090) + WebDAV (:9091)
```

## 五、加密系统

### 5.1 加密方案

| 项目 | 值 |
|------|-----|
| 算法 | AES-256-GCM（认证加密） |
| 密钥派生 | PBKDF2-SHA256, 100,000 次迭代 |
| 密钥长度 | 32 字节 |
| Nonce | 12 字节随机（文件）/ 确定性（名称） |
| GCM Tag | 16 字节认证标签 |

### 5.2 文件内容加密

```
加密格式:
┌────────────────────────────────────────────┐
│ 12B Nonce │     Ciphertext     │ 16B GCM Tag │
│ (随机)    │ (AES-256-GCM 加密)  │ (认证标签)   │
└────────────────────────────────────────────┘
总开销: 每个文件增加 28 字节
```

### 5.3 密钥派生

```
password (用户密码)
    ↓
PBKDF2-SHA256(password, salt, 100000, 32)
    ↓
32 字节 AES-256 密钥
    ↓
AES-GCM 密钥（内容加密 + 名称加密共用）
```

### 5.4 SQLite 元数据 Schema

```sql
CREATE TABLE files (
    cloud_file_id   TEXT PRIMARY KEY,    -- 和彩云文件 ID
    parent_cloud_id TEXT NOT NULL,        -- 父目录 ID
    cloud_name      TEXT NOT NULL,        -- 云端名 (E_xxx.bin)
    orig_name       TEXT NOT NULL,        -- 原始名 (report.pdf)
    orig_path       TEXT NOT NULL,        -- 原始完整路径
    type            TEXT NOT NULL,        -- "file" 或 "folder"
    orig_size       INTEGER NOT NULL,     -- 明文大小
    enc_size        INTEGER NOT NULL,     -- 密文大小
    nonce           BLOB,                -- GCM nonce
    content_hash    TEXT NOT NULL,        -- 明文 SHA256
    is_encrypted    INTEGER NOT NULL,     -- 是否加密
    created_at      DATETIME,
    updated_at      DATETIME
);

CREATE TABLE kv_store (
    key   TEXT PRIMARY KEY,
    value TEXT NOT NULL
);
```

## 六、名称加密

### 6.1 概述

**文件名和文件夹名在云端完全不可辨别。** 任何人直接访问和彩云都只能看到不可读的 hex 字符串。

### 6.2 加密规则

| 类型 | 原始名 | 云端名 |
|------|--------|--------|
| 文件夹 | `Documents` | `E_b28ce2c86ef5804d326dd8fbfe5ffe72...` |
| 文件 | `report.pdf` | `E_50db1161a60377cec9ee89d132dde32f...bin` |
| 文件 | `secret.txt` | `E_2566249bcadbf4ee9520d3eacfe2aef5...bin` |

**云端看到的目录结构:**
```
/MopanProxy/
├── E_b28ce2c86ef5804d326dd8fbfe5ffe726d8d51250b5d70077a2055ebd258440d...  ← 无法知道这是什么
│   └── E_2566249bcadbf4ee9520d3eacfe2aef5cd363ff6ab22d87cacf09e6f2d42...bin  ← 无法知道文件名
├── E_50db1161a60377cec9ee89d132dde32fee5dfa9932ad88d13eaa0ed5d5c1d9cc...bin  ← 无法知道文件名
└── Documents_enc  ← 旧版兼容
```

**用户看到的（通过 MopanProxy）:**
```
/MopanProxy/
├── SecretDocs/
│   └── conf.txt
├── secret.txt
└── Documents/  ← 旧版文件夹仍可访问
```

### 6.3 加密算法

```go
// 名称加密: AES-256-GCM + 确定性 nonce
EncryptName(name):
    nonce = SHA256(name + "_name_nonce")[:12]  // 相同名称 → 相同 nonce → 相同密文
    ciphertext = AES-GCM-Encrypt(key, nonce, name)
    return hex(nonce + ciphertext + tag)       // 40 字节 = 80 hex

// 文件夹: "E_" + hex
EncNameForFolder(name) = "E_" + EncryptName(name)

// 文件: "E_" + hex + ".bin"
EncNameForFile(name) = "E_" + EncryptName(name) + ".bin"
```

### 6.4 名称还原

```
云端名 → SQLite 查 orig_name → 显示给用户

示例:
E_50db1161...bin → SQLite lookup → "secret.txt"
E_b28ce2c8...    → SQLite lookup → "SecretDocs"
Documents_enc    → 旧版兼容     → "Documents" (去掉 _enc)
report.pdf.enc   → 旧版兼容     → "report.pdf" (去掉 .enc)
```

### 6.5 安全特性

| 特性 | 说明 |
|------|------|
| **文件名不可读** | 云端只看到 `E_` + hex |
| **文件类型隐藏** | 所有文件后缀统一为 `.bin` |
| **内容加密** | AES-256-GCM 认证加密 |
| **篡改检测** | GCM 认证标签检测任何修改 |
| **密钥不存储** | 从密码派生，内存中不留明文 |
| **确定性名称** | 相同文件名总是加密为相同密文 |

## 七、和彩云 API

### 7.1 API 域名

| 用途 | 域名 |
|------|------|
| 文件操作 | `https://personal-kd-njs.yun.139.com` |
| 用户/路由 | `https://user-njs.yun.139.com` |

### 7.2 文件操作 API

| 功能 | 方法 | 路径 |
|------|------|------|
| 文件列表 | POST | `/hcy/file/list` |
| 创建文件/目录 | POST | `/hcy/file/create` |
| 获取下载链接 | POST | `/hcy/file/getDownloadUrl` |
| 删除文件 | POST | `/hcy/file/batchDelete` |
| 移动文件 | POST | `/hcy/file/batchMove` |
| 复制文件 | POST | `/hcy/file/batchCopy` |
| 更新文件信息 | POST | `/hcy/file/update` |
| 搜索文件 | POST | `/hcy/file/search` |
| 完成上传 | POST | `/hcy/file/complete` |

### 7.3 上传流程（三步确认）

```
Step 1: POST /hcy/file/create
  请求: {parentFileId, name:"E_xxx.bin", size, contentHash, partInfos}
  响应: {fileId, partInfos[{uploadUrl}]}

Step 2: PUT {uploadUrl}
  请求体: 密文二进制数据
  响应: 200 OK

Step 3: POST /hcy/file/complete
  请求: {fileId, uploadId, contentHash(密文的hash), contentHashAlgorithm}
  响应: {success: true}

⚠️ 缺少 Step 3 会导致文件不显示！
```

## 八、API 接口文档

### 8.1 基础信息

- Base URL: `http://your-server-ip:9090`
- Content-Type: `application/json`
- **所有 POST 请求需要 `X-Requested-With: XMLHttpRequest` 头（CSRF 保护）**

### 8.2 安全机制

| 机制 | 说明 |
|------|------|
| 认证中间件 | 除 `/api/status` 外所有端点需登录 |
| CSRF 保护 | POST 请求需要 `X-Requested-With` 头 |
| 请求体限制 | JSON 请求最大 1MB |
| 安全头 | HSTS, CSP, X-Frame-Options, X-Content-Type-Options |

### 8.3 接口列表

#### GET /api/status — 状态检查（无需认证）

```json
{
  "success": true,
  "data": {
    "auth": true,
    "account": "13800138000",
    "encrypted": true,
    "root_id": "Fu72B2..."
  }
}
```

#### POST /api/set-token — 设置 Token

请求:
```json
{"authorization": "Basic cGM6MTk4..."}
```

#### GET /api/files?path=/ — 文件列表

参数:
- `path`: 目录路径（默认 `/`）
- `cursor`: 分页游标

响应:
```json
{
  "success": true,
  "data": {
    "files": [
      {
        "id": "Fxxx",
        "name": "SecretDocs",
        "size": 0,
        "type": "folder",
        "created_at": "2026-06-20T...",
        "updated_at": "2026-06-20T...",
        "is_encrypted": true
      }
    ],
    "next_cursor": "",
    "total": 1
  }
}
```

#### POST /api/upload — 上传文件

请求 (multipart/form-data):
- `file`: 文件内容
- `path`: 目标路径（默认 `/`）

响应:
```json
{
  "success": true,
  "data": {
    "message": "上传成功",
    "encrypted": true,
    "orig_size": 1024,
    "enc_size": 1052
  }
}
```

#### GET /api/download/{fileId} — 下载文件

响应: 文件内容（加密文件自动解密）

#### POST /api/mkdir — 创建文件夹

请求:
```json
{"parent_id": "/", "name": "Documents"}
```

#### POST /api/delete — 删除文件

请求:
```json
{"file_ids": ["Fxxx", "Fyyy"]}
```

#### GET /api/stats — 加密统计

```json
{
  "success": true,
  "data": {
    "encrypted_files": 42,
    "total_orig_size": 1073741824,
    "total_enc_size": 1073741870,
    "encryption": true,
    "root_folder": "/MopanProxy",
    "root_id": "Fu72B2..."

```

#### GET /api/settings — 系统设置（密码已掩码）

```json
{
  "success": true,
  "data": {
    "account": "13800138000",
    "webdav_port": 9091,
    "webdav_user": "admin",
    "webdav_pass": "****",
    "encryption_enabled": true,
```

## 十一、安全机制

### 11.1 认证与授权

| 机制 | 说明 |
|------|------|
| 全局认证中间件 | 所有 `/api/*` 端点（除 status）需认证 |
| WebDAV Basic Auth | 独立用户名/密码 |
| CSRF 保护 | POST 需要 `X-Requested-With` 头 |
| 密码掩码 | Settings API 不返回明文密码 |

### 11.2 传输安全

| 机制 | 说明 |
|------|------|
| HSTS | `max-age=31536000; includeSubDomains` |
| CSP | `default-src 'self'; script-src 'self' 'unsafe-inline'` |
| X-Frame-Options | `DENY` (禁止 iframe 嵌入) |
| X-Content-Type-Options | `nosniff` |
| HTTPS | 通过 Cloudflare 提供 TLS |

### 11.3 数据安全

| 机制 | 说明 |
|------|------|
| 内容加密 | AES-256-GCM (认证加密) |
| **名称加密** | **文件名在云端不可辨别** |
| 密钥不存储 | PBKDF2 从密码派生 |
| 配置文件权限 | 0600 (仅 root 可读) |
| 请求体限制 | JSON 最大 1MB |

### 11.4 运行时安全

| 机制 | 说明 |
|------|------|
| 优雅关闭 | SIGTERM → HTTP Shutdown + SQLite Close |
| 错误信息 | 不泄露内部路径 |
| 并发安全 | config.Save 使用互斥锁 |
| 重试机制 | 轮询替代 time.Sleep |

### 11.5 WebUI 安全

| 机制 | 说明 |
|------|------|
| XSS 防护 | 事件委托 + data-* 属性，无 inline onclick |
| 属性转义 | `escAttr()` 转义 HTML 属性字符 |
| CSRF header | 所有 XHR 请求带 `X-Requested-With` |

## 十三、故障排除

### 13.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 401 未登录 | Token 过期 | 重新获取 Token |
| 403 CSRF 验证失败 | 缺少 X-Requested-With 头 | 添加请求头 |
| 上传不显示 | 缺少 /file/complete | 已修复 |
| 下载 500 错误 | Token 过期或文件不存在 | 检查 Token |
| 加密解密失败 | 密码错误 | 检查 config.yaml |
| 名称无法解密 | 换了密码/salt | 恢复原 config.yaml |
| WebDAV 405 | PUT 方法不支持 | 检查配置 |
